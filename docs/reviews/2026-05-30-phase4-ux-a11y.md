## What changed (Phase 4)
- **a11y semantics:** chat timeline `role="log"`+`aria-live` (agent replies announce); auth mode toggle → WAI-ARIA `tablist`/`tab`/`tabpanel`; create-room `role="dialog"`+`aria-modal`+labelled; `<main>` landmark; `role="status"`/`role="alert"` on loading/empty/error states; labelled nav/aside/select; Toast escalates to `alert`/assertive for errors.
- **theming/contrast:** theme-aware inline-code + code-block rendering via `color-mix` (replaced `bg-black/10`/`bg-white/65` literals that were illegible on light themes); fixed real WCAG-AA `--muted` contrast bugs (light-modern, solarized-light).
- **motion:** `prefers-reduced-motion` guard (CSS) + JS `scrollIntoView` branch.
- **automation:** new `e2e/a11y.spec.ts` runs axe-core (WCAG 2a/2aa/21a/21aa) on `/auth` in both modes; fails on any serious/critical violation. CI runs it via `playwright test`.
- **lint:** 29 → 7 warnings, 0 errors (disabled non-type-aware core `no-unused-vars`; removed dead `WORKER_ID`; documented SSR cookie empty-catch).

The 4 Lows were all addressed (see below).

### accessibility-tester → raised 4 "Critical" + 5 "High"
Evaluated with the `receiving-code-review` discipline (verify before implementing). Several were **mis-severitised** against WCAG (the buttons were already keyboard-operable; arrow-keys/focus-trap are APG *authoring practices*, and the toast-timing item is Level AAA). The genuinely valuable items were fixed; the rest are justified deferrals.

**Fixed (commit `b551802`):**
- **Tab keyboard nav** — added roving `tabindex` + Left/Right/Home/End arrow navigation to the auth tablist; new e2e (`tablist supports arrow-key navigation`) proves selection + focus move.
- **Modal focus management** — focus trap (Tab/Shift+Tab cycle), return-focus-to-trigger on close, backdrop click-to-dismiss (`target===currentTarget`, so in-dialog clicks don't close).
- **Nested live regions** — removed `role="status"` from the empty state and the AgentRunCard status row (both inside the `role="log"` region → would double-announce). Standalone LoadingSkeleton keeps `role="status"`+explicit `aria-live`.
- **Error contrast** — `text-red-600` → `text-red-700` on `red-50` (auth + sidebar); red-600/red-50 was ~4.4:1, under AA 4.5:1.
- **Reduced-motion + JS scroll** — `scrollIntoView({ behavior: reduced ? 'auto' : 'smooth' })`.

**Deferred (justified):**
- *Toast auto-dismiss ignores reduced-motion* — WCAG 2.2.3 is **Level AAA** (not AA); pre-existing behavior. Tracked for a later pass.
- *Disabled-select opacity* — reviewer cited 1.3.1, but dim disabled controls are not an Info-and-Relationships failure; the control already has an `aria-label`.
- *`0.01ms` reduced-motion value* — the widely-used community snippet value; speculative jank concern, no evidence on target browsers.
