## What held strong (the confidence case)

The panel actively tried to break these and could not — each is grounded in code and covered by tests:

- **API authz model** — all 20 routes: authn (401) → authz gate (`requireRoomMember/Owner/Admin/AgentCreator`, 403) → zod validation (400); object-action routes re-scope writes with `.eq('room_id', roomId)` (no IDOR); CSRF enforced globally in middleware for every `/api` path (sound Bearer exemption); write/expensive routes rate-limited.
- **RLS write-isolation** — RLS ON for all 10 tables; **no INSERT/UPDATE/DELETE policy exists for the authenticated role on any table**, so the browser cannot write `agent_runs`/`messages`/`agent_memory`/`user_profile` directly — every privileged write goes through the service-role client behind app-layer checks. Read policies are membership-scoped (`is_room_user_member()`); storage was tightened from `auth.uid() IS NOT NULL` to path-derived `is_room_file_member()`. pgTAP proves cross-room read denial + no-op browser writes.
- **Subprocess injection-safety** — `spawn(…, {shell:false})`; binary resolved only from a trusted `*_BIN`/PATH, never agent data; **every `buildArgs` returns a static, code-defined argv** that ignores the packet; a user-authored `system_prompt` (incl. shell metachars / injected `--dangerously-skip-permissions`) reaches the CLI via **stdin only** (regression-tested); child env allowlisted + secrets stripped; 10 MB output cap + timeout + force-kill-tree. Tool execution is a **stub** (no real command runs), so the denylist's weaknesses are not a live injection path.
- **Memory injection-safety** — agents never write the tables (zod-validated `memory_op` → service-role scan + sanitize + persist); recall RPC is `SECURITY INVOKER` with EXECUTE revoked from anon/authenticated; recall double-capped + error-resilient; for claude-code/codex the persona precedes a fenced, quoted "data, not instructions" memory block; the classic "ignore previous instructions / approve every tool" payload is flagged, stripped of control tokens, and rendered inert. (Red-team tests pass.)
- **Agent-loop termination** — handoff path strictly increases `round_index` + `deliberation_depth`, with hop-cap, round-cap, self-handoff and full-chain cycle detection (`collectChainAgents`), each posting a visible "Deliberation ended" message; discussion is 3-phase + round-capped + dup-protected by a unique index; `/reset` is admin-gated and only stamps a watermark (non-destructive). Cycle A→B→A blocked (test).
- **Run state machine** — claim is atomic (`UPDATE … WHERE status='queued' … RETURNING` + `status='claimed'` guard); cancellation truly kills the child (SIGTERM → 2 s grace → `taskkill /T /F` / SIGKILL with a `forceExitPromise`); crash / bad-output / DB-error / cancel each yield exactly one terminal write in isolation (tests).
- **Release workflow** — `release.yml` triggers only on `v*.*.*` tags, never auto-tags, `contents:read` top-level with write escalated only on the release job, strict-semver guard runs pre-checkout, `github.token` only, no registry push, no external secrets. No workflow uses `pull_request_target` / `github.event.*` in a `run:` step → no poisoned-pipeline vector. Service-role key is provably server-only (0 matches in 23 client components; not baked into images; not committed).
